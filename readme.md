## Scrum Board Workflow
Our GitHub Project board includes the following lanes:
1. To Do  
2. In Progress  
3. Code Review  
4. Testing  
5. Done  

## Sprint Process
- Sprint length: 1–2 weeks  
- Sprint Planning: Product Owner selects PBIs for the sprint backlog  
- Daily Scrum:
  - Scrum Master asks: “Are there any blockers?”  
  - Developers respond with status updates or blockers  
- Issue flow:
  - In Progress → Code Review → Testing → Done  

## Definition of Done
A Product Backlog Item (PBI) is considered complete when:
- Development is finished  
- Code has been reviewed by a different developer  
- Feature has been tested  
- Issue is moved to the "Done" lane  

## Sprint Tracking
- Each issue is labeled by sprint (e.g., Sprint 1, Sprint 2)  
- Issues are reassigned to a different developer at each stage (development, review, testing)
