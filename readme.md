## Scrum Process Overview

We follow Scrum methodology using GitHub Projects to manage our sprint workflow.  
All Product Backlog Items (PBIs) are tracked as GitHub Issues and move across the board lanes.

---

## GitHub Project Board

Our board contains the following lanes:
- To Do
- In Progress
- Code Review
- Testing
- Done

All issues are linked to the project board and follow this workflow.

## Issue Workflow (PBI Lifecycle)

Each issue moves through the following stages:

1. **To Do** → Issue is created and ready for work  
2. **In Progress** → Assigned developer begins implementation  
3. **Code Review** → Assigned to a different developer for review  
4. **Testing** → Assigned to a different developer for testing  
5. **Done** → Work is completed and verified  

- A **different developer is assigned at each stage**
- Status changes reflect ownership changes

---

## Daily Scrum (Standup Evidence)

For every issue in:
- In Progress  
- Code Review  
- Testing  

The following comments are added:

- Scrum Master: “Are there any blockers?”  
- Developers: “No blockers” or explanation of blockers  

This occurs at each team meeting.

---

## Sprint Management

- Work is organized into **1–2 week sprints**
- Each issue is labeled by sprint:
  - Sprint 1
  - Sprint 2
  - Sprint 3

- Sprint Planning defines:
  - Sprint backlog
  - Work selected by Product Owner  

---

## Definition of Done

A PBI is considered **Done** when:
- Development is complete  
- Code review is completed by another developer  
- Testing is completed by another developer  
- Issue is moved to the Done lane  

---

## Commit and Workflow Alignment

- Commits are made throughout development and reflect progress on issues  
- Commit history aligns with issue status changes  
- Issues move gradually (not instantly) from In Progress to Done  

---

## Final Notes

- All issues are in either:
  - To Do  
  - Done  

- No issues remain stuck in intermediate states at sprint end  
- All issues are assigned and tracked through the full Scrum lifecycle
